# Code Of Conduct

Dox aims to be a professional, focused, and respectful open-source project.

## Our Standards

Examples of behavior that helps the community:

- using welcoming and respectful language;
- giving and receiving technical feedback in good faith;
- focusing on the project goal instead of personal attacks;
- explaining tradeoffs, risks, and constraints clearly;
- respecting different levels of familiarity with the Amazon ecosystem, Go, frontend engineering, data systems, and operations.

Examples of unacceptable behavior:

- harassment, insults, threats, or discriminatory language;
- sexualized language or imagery;
- publishing someone else's private information without permission;
- exposing credentials, private business data, or sensitive logs;
- repeatedly derailing issues or pull requests away from the project scope.

## Enforcement

Maintainers may edit, hide, reject, or remove comments, issues, pull requests, commits, and other contributions that do not follow this code of conduct.

Serious or repeated violations may result in temporary or permanent restrictions from participating in the project.

## Reporting

You can report conduct concerns by contacting `frostleo.dev@gmail.com`.

Security vulnerabilities should be reported according to [SECURITY.md](SECURITY.md), not through public issues.
